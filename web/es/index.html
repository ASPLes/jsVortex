¿Qué es jsVortex?

jsVortex es una implementación Open Source del protocolo "Block Extensible Exchange Protocol (BEEP)", conocido como "Beepcore", principalmente definido en los documentos RFC3080 y RFC3081. jsVortex proporciona una base extensible para protocolos de aplicación, para producir aplicaciones de internet altamente interactivas sobre la plataforma WEB.

No es de ningún modo un reemplazo completo del protocolo HTTP, sino un complemento, y en algunos casos una alternativa, que es útil donde se requieren "más" características y una "nueva" forma de intercambiar datos entre el servidor y el navegador.

Actualmente Firefox es el único navegador soportado, pero se espera poder sopotar otros navegadores en el futuro.

jsVortex fue escrito por ASPL, desarrolladores de Vortex Library y Turbulence.

¿Qué es BEEP?

BEEP (http://www.beepcore.org) es un poderoso protocolo de aplicación que le permite diseñar la mejor solución para sus necesidades de intercambio y transferencia de datos. Para ello, BEEP le proporciona las mejoras prácticas y características que son "siempre" necesitadas por cualquier aplicación de red:

  • Soporte integrado de autenticación a través de SASL.
  • Soporte para sessiones seguras con TLS.
  • Intercambio asíncrono de datos: ambos puntos de red (o "peers") pueden enviar o recibir un mensaje BEEP en cualquier momento.
  • Soporte para canales independientes lo que permite una separación y una mejor reutilización de varios protocolos de aplicación sobre una misma sessión BEEP.
  • Disponibles varios tipos de mensaje y patrones de intercambiio: MSG, RPY, ERR and ANS/NUL.
  • Soporte integrado de MIME.
  • Soporte integrado para la negociación en tiempo real de perfiles a utilizar, lo que permite desplegar varias versiones distinas de un protocolo sobre la misma sessión BEEP sin problemas, o mezclar distintos protocolos o implementar un soporte real de virtual hosting sobre la misma IP para el protocolo TLS.
